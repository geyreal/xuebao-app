# 学伴小书包 APP 需求设计

## 一、项目总述

### 1. 项目名称

学伴小书包 APP

### 2. 开发模式

覆盖 Android 端、iOS 端、后台管理系统（网页端）。

### 3. 核心目标

聚焦语文、数学、英语、综合四大学科类别，实现学科学习资源线上化沉淀；核心实现班级权限隔离，支持教师按班级开展资源管理、打卡布置，学生按班级进行自主学习、打卡反馈，满足校内轻量化教学辅助、学生自主学习、教师打卡监督的核心需求，杜绝跨班级数据干扰，适配校内日常教学场景。

### 4. 用户角色

- **学生**：绑定个人班级，查看本班专属学习资源，支持在线阅读文档、收听音频，完成本班打卡任务，查看个人学习及打卡记录。
- **教师**：绑定所教班级，上传、管理本班学科资源，发布本班打卡任务，查看本班学生打卡数据（打卡率、已打卡/未打卡名单），实现班级学习闭环管理。
- **管理员**：负责全校账号（学生、教师）管理、班级信息管理、学科分类管理、全校数据总览及权限分配；处理违规资源，保障系统正常运行。

## 二、功能需求设计

### （一）公共基础功能

#### 1. 登录与身份认证

- **账号密码登录**：采用学号（学生）/ 工号（教师/管理员）作为账号，支持自定义密码，密码加密存储。
- **密码管理**：支持密码重置（通过管理员后台重置，学生/教师可联系管理员操作）、自动登录（可选，切换 APP 后无需重复输入密码）。
- **角色自动识别**：登录时系统根据账号类型，自动识别学生、教师、管理员角色，跳转至对应角色首页，无权限越界。
- **班级绑定**：登录后自动绑定用户预设的年级+班级，学生无法切换班级，教师仅显示所负责班级，管理员可查看所有班级。

#### 2. 全局通用功能

- **全局搜索**：支持按资源标题、学科进行模糊搜索，仅展示当前用户权限范围内（本班/全校）的相关资源。
- **消息通知**：推送资源更新提醒（教师上传新资源后，本班学生收到通知）、打卡任务提醒（打卡任务发布、截止前 1 天/半天提醒），通知可在消息中心查看历史记录。
- **UI 设计**：采用简洁、清爽的界面风格，无任何广告、无会动画，字体清晰、操作便捷，适配学生使用习惯，降低学习操作门槛。

### （二）四大学科模块（语文/数学/英语/综合组）

#### 1. 学生端功能

- **资源查看**：仅能查看自己所在班级的四大学科资源，无法查看其他班级资源，资源按学科、上传时间排序。
- **支持格式**：支持查看图片（知识点图文、练习题图片等）、PDF 文档（课件、练习题、知识点总结等）、MP3 音频（英语听力、语文课文朗诵等）。
- **文档预览**：支持 PDF、Word 文档在线预览，可实现缩放、翻页、横竖屏自动适配，满足不同阅读场景需求。
- **音频播放**：支持 MP3 音频播放、暂停、倍速调节（0.5x-2.0x）、进度拖拽调节，支持后台播放（切换 APP 后音频不中断）。
- **学习辅助**：支持资源收藏（收藏后可在个人中心快速查看）、学习记录标记（标记已学习/未学习），方便后续继续学习。
- **打卡关联**：学习完成对应资源后，可直接跳转至本班打卡页面，完成打卡操作，实现学习与打卡的无缝衔接。

#### 2. 教师端功能

- **资源管理**：仅能对自己负责的班级上传、管理资源，无法操作其他班级资源，保障班级资源独立性。
- **资源上传**：上传时需填写资源标题、所属学科、年级、对应班级、资源文件、资源简介（简要说明资源内容，方便学生了解）。
  - 支持格式：PDF、Word（需兼容主流版本，避免格式错乱）、图片（JPG/PNG 格式，清晰无模糊）、音频（MP3 格式，音质清晰、无杂音）。
- **资源操作**：支持对已上传资源进行编辑（修改标题、简介、所属班级等）、删除、置顶（置顶后在班级资源列表顶部显示）、下架（下架后本班学生无法查看）。
- **权限控制**：上传的资源仅对指定班级可见，其他班级无法查看，确保班级资源的私密性和针对性。

### （三）核心：班级权限隔离规则

班级权限隔离是本 APP 核心需求，所有功能、数据均围绕此规则设计，确保不同班级数据完全独立，杜绝越权访问和数据干扰。

- **学生权限**：仅能查看、访问自己所在班级的所有资源、打卡任务、学习记录，无法查看其他班级任何数据。
- **教师权限**：仅能管理自己负责的班级（上传/编辑/删除本班资源、发布本班打卡任务、查看本班学生打卡数据），无法操作其他班级数据。
- **数据隔离**：不同班级之间的资源、打卡任务、学习记录、打卡数据完全独立、互不干扰，确保班级数据的针对性。
- **管理员权限**：拥有最高权限，可查看、管理全校所有班级、教师、学生账号及相关数据，可分配教师与班级的对应关系，处理违规资源。
- **数据锁定**：所有资源、打卡任务、学习记录、打卡数据绑定唯一班级 ID，通过班级 ID 实现权限过滤，确保数据隔离的有效性。

### （四）学习打卡模块（按班级隔离）

打卡模块与班级权限深度绑定，仅展示当前用户所在班级（学生）或所负责班级（教师）的打卡相关数据，实现精准打卡管理。

#### 1. 学生端

- **任务查看**：登录后可直接查看本班今日打卡任务、未完成打卡任务、已完成打卡任务，明确打卡要求和截止时间。
- **一键打卡**：学习完成后，可在打卡页面点击"一键打卡"完成操作，打卡功后标记为已完成，无法重复打卡。
- **打卡记录**：支持查看个人打卡日历（直观显示每日打卡状态）、连续打卡天数（鼓励学生坚持学习）、历史打卡记录（可查看过往打卡详情）。
- **数据展示**：仅展示本班相关打卡数据，不显示其他班级打卡情况，确保数据针对性。

#### 2. 教师端

- **任务发布**：可给所负责班级发布打卡任务，需设置任务名称、所属学科、打卡截止时间（可选，明确打卡要求）、任务说明（可选，明确打卡要求）。
- **数据查看**：查看本班打卡数据，包括打卡率（已打卡人数/班级总人数）、已打卡名单、未打卡名单，可快速掌握学生打卡情况。
- **统计功能**：支持简单数据统计，无需复杂导出功能，可直观查看本班打卡整体情况，方便教师进行督促。

### （五）个人中心

个人中心按角色展示对应功能，数据均为当前用户权限范围内的内容。界面简洁，操作便捷。

- **个人信息**：展示用户姓名、班级、角色（学生/教师/管理员），信息不可随意修改，需通过管理员后台调整。
- **我的收藏**：展示用户收藏的所有资源，可快速查看、取消收藏，支持按学科筛选。
- **我的学习记录**：展示用户所有学习记录（已学习/未学习资源），可按时间、学科排序，方便后续回顾学习。
- **我的打卡数据**：展示用户个人打卡情况（连续打卡天数、打卡完成率、历史打卡记录）。学生端展示个人数据，教师端展示所教班级整体打卡数据。
- **系统设置**：支持密码修改、APP 缓存清理、查看 APP 版本信息。管理员端额外增加系统配置入口。

### （六）管理后台（网页端）

管理后台为网页端，支持管理员通过电脑操作，功能聚焦全校管理、操作简洁，适配单人开发维护需求。

#### 1. 班级管理

- **班级操作**：支持添加新班级（填写年级、班级名称）、编辑班级信息、删除无用班级（需确认无关联学生/教师后可删除）。
- **教师分配**：分配教师与班级的对应关系，一个教师可负责多个班级，一个班级可对应多个教师（如不同学科教师）。

#### 2. 用户管理

- **账号管理**：批量添加、编辑、删除学生、教师账号，支持批量导入账号信息（简化操作，适配校内多用户场景）。
- **账号操作**：为学生/教师绑定对应班级，支持密码重置（学生/教师忘记密码时使用）、账号启用/禁用（处理违规账号或毕业学生账号）。

#### 3. 资源总管理

- **资源查看**：查看全校所有班级、所有学科的上传资源，可按班级、学科、上传时间筛选。
- **违规处理**：对违规、不适宜的资源进行下架处理，下架后对应班级学生无法查看，同时可通知对应教师。

#### 4. 数据总览

- **打卡统计**：查看全校各班级打卡情况（打卡率、平均打卡完成率），可按年级、班级筛选，直观掌握全校打卡整体情况。
- **资源统计**：统计各学科资源使用量（查看次数、收藏次数），了解各学科资源使用情况，为教学优化提供参考。

#### 5. 系统设置

- **公告发布**：发布全校公告（如 APP 使用说明、教学通知等），所有用户登录后可在首页查看公告。
- **基础配置**：配置 APP 基础参数（如资源上传大小限制、打卡提醒时间等），保障系统稳定运行。

## 三、APP 页面结构框架（最简结构）

页面设计遵循"简洁实用、操作便捷"原则，减少冗余页面，降低开发难度，适配单人开发节奏，同时满足核心功能使用需求。

### 底部导航（4 个，全局固定）

| 导航项 | 说明 |
|--------|------|
| 首页 | 展示公告、本班学科入口、今日打卡提醒、班级提示（当前绑定班级） |
| 学科中心 | 四大学科切换入口，展示本班各学科资源列表 |
| 打卡中心 | 展示本班打卡任务、个人打卡记录、打卡日历 |
| 我的 | 个人中心，包含个人信息、收藏、学习记录、打卡数据、系统设置 |

### 主要页面（按使用流程排序）

| 页面 | 说明 |
|------|------|
| 登录页 | 账号密码输入框、密码重置入口、自动登录选项，登录后跳转至对应角色首页 |
| 首页 | 公告展示区、学科快速入口（四大学科）、打卡提醒（未完成任务）、班级提示，教师端额外增加"工作台"入口 |
| 学科列表页 | 四大学科切换按钮，资源按上传时间排序，支持搜索、筛选，展示资源标题、简介、上传时间 |
| 资源详情页 | 文档/音频播放主体，收藏、标记学习状态按钮，返回、分享（仅分享至本班）功能 |
| 打卡页 | 今日打卡任务、打卡按钮、打卡日历、历史打卡记录，教师端展示本班打卡统计数据 |
| 个人中心 | 个人信息展示、我的收藏、我的学习记录、我的打卡数据、密码修改、缓存清理、版本信息 |
| 教师工作台 | 资源上传入口、本班资源管理、打卡任务发布、本班打卡数据查看 |
| 管理员后台（网页端） | 班级管理、用户管理、资源总管理、数据总览、系统设置，各功能模块独立分区 |

## 四、非功能需求（可实现基本标准）

| 需求类别 | 说明 |
|----------|------|
| 兼容性 | 支持 Android 8.0 及以上版本、iOS 12.0 及以上版本，适配手机、平板两种机型设备，横竖屏自动适配 |
| 性能 | APP 启动时间 ≤3 秒，资源加载时间 ≤5 秒（网络正常情况下），音频播放无卡顿、无杂音，后台播放稳定 |
| 安全性 | 账号密码加密存储，防止信息泄露；严格执行班级数据隔离，杜绝越权访问；仅面向校内用户使用，不对外开放 |
| 存储 | 采用云端存储（选择低成本、稳定的云存储服务），支持资源本地缓存（节省流量，离线可查看已缓存资源），缓存可手动清理 |
| 稳定性 | 支持校内 500-2000 人同时使用，无崩溃、卡顿现象，服务器响应稳定，满足校内日常使用峰值需求 |
